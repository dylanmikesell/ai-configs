# Rebuilding `ngi-pptx` when the corporate template changes

NGI refreshes the PowerPoint template every couple of years. This skill is built so a
refresh is a ~30-minute mechanical update, not a rewrite. Read this before updating;
then hand it (and the new template) to Claude and say *"update the ngi-pptx skill for
the new template following REBUILD.md"* — it can do almost all of it.

## Why the update is easy — the core strategy

The skill **never hard-codes NGI branding.** `SKILL.md` and `scripts/pptx_helpers.py`
*open the template file and add slides from its own layouts*, so fonts, colours, slide
size, and master graphics are **inherited** — swap the file and they update themselves.

That means only two kinds of things actually have to be touched on a refresh:

1. **The bundled template file** — replace it.
2. **The human-readable references derived from it** — the layout map
   (`references/layouts.md`) and the visual gallery
   (`references/layout-gallery.pptx`). Both are regenerated by scripts.

The skill's **only real coupling** to the template is **layout names** — it matches
layouts by exact name. If NGI renames layouts or switches their language, that's the
main thing to fix. Everything else is inherited or regenerated.

## Coupling points to check between versions

| What can change | Where it shows up | Inherited or edit? |
|---|---|---|
| Slide size (e.g. → true 16:9) | page dimensions | inherited; update the figure in `layouts.md` + `SKILL.md` premise |
| Theme fonts (Aptos → …) | headings/body | inherited; update the reference lines |
| Palette hex values | colours | inherited; update palette lists in `layouts.md` + `SKILL.md` |
| **Layout names** | layout selection | **must edit** — the skill matches by name |
| **Placeholder indices** | where text/pictures go | **must edit** `layouts.md` (and any hard-coded idx in examples) |
| Layouts added/removed | the whole map | **must edit** map + gallery + selection table |

## Step-by-step

Prereq: `python-pptx` installed (`pip install python-pptx`).

1. **Replace the template.** Save the new corporate file over
   `assets/ngi-template.pptx` — **keep the same filename** so nothing else needs
   repathing. (The helpers resolve it by that name.)

2. **Inspect it.** Dump every branding fact as JSON:
   ```
   python scripts/inspect_template.py > new.json
   ```
   This lists slide size, theme fonts, theme colours, company (sanity-check it says
   NGI), and every layout with its exact name and placeholder idx/type. It uses only
   the standard library, so it never depends on the thing it inspects.

3. **Regenerate `references/layouts.md`** from `new.json`:
   - Update the size / fonts / palette lines at the top.
   - Rebuild the table: exact layout names, placeholder indices, and re-do the English
     gloss + "use for" columns (translate any Norwegian). **Flag name quirks** — the
     old template had a layout named `" Tidslinje2"` with a leading space; such quirks
     break name matching if not copied verbatim.
   - Fastest path: give `new.json` to Claude → "regenerate layouts.md for this template".

4. **Regenerate the gallery:**
   ```
   python scripts/make_gallery.py
   ```
   Open `references/layout-gallery.pptx` in PowerPoint and eyeball every layout.

5. **Update `SKILL.md`** everywhere it names specifics:
   - the *Choosing a layout* table (layout names),
   - the example code (layout names / placeholder indices used),
   - the *Fonts & colours* block,
   - the slide-size figure in the opening premise, if it changed.

6. **Re-validate the workflow** (don't trust it opens — check):
   - `describe(open_template())` prints the new layouts + placeholders — spot-check
     against `layouts.md`.
   - Build a quick deck (cover + two-column + image) using the **new** layout names,
     save it, then:
     ```
     python scripts/check_integrity.py <deck>.pptx
     ```
     Must report no duplicate parts and open cleanly. Also open it once in PowerPoint
     to confirm no "repair" prompt.

7. **Commit & push** to `ai-configs`
   (`feat(ngi-pptx): update for the <year> corporate template`).

8. **Re-deploy to the desktop app / claude.ai.** A git push does **not** reach that
   Skills system. Re-zip the `ngi-pptx` folder and re-upload it under
   **Settings → Capabilities → Skills**.

9. **Update the AI MOC** (`Extras/MOC/🤖 AI + Claude MOC.md`) — new date and any
   changed layout names — per the standing "document new AI capabilities" rule.

## The tools in `scripts/`

| Script | Role in a rebuild |
|---|---|
| `inspect_template.py` | Step 2 — dump all template facts as JSON (stdlib only, any .pptx) |
| `make_gallery.py` | Step 4 — regenerate the visual layout gallery |
| `check_integrity.py` | Step 6 — assert a produced deck has no duplicate parts and opens |
| `pptx_helpers.py` | The runtime API; `describe(open_template())` for a live layout dump |

## Guardrails to preserve in any rebuild

- **Never hard-code fonts/colours/size in code** — keep deferring to the template. That
  is the whole reason refreshes are cheap.
- **Match layouts by name, not index** — indices are more volatile than names.
- **Keep the delete-with-relationship fix in `blank_deck()`.** Removing a slide's id
  without also dropping its relationship reintroduces the duplicate-part corruption that
  triggers PowerPoint's repair prompt.
- **Keep filenames stable** (`assets/ngi-template.pptx`) so the swap is a one-file
  replace.
